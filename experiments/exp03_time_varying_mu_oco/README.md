# Experiment 03: Time-Varying 1D Quadratic OCO for SMART

## Why this experiment exists

This experiment is the cleanest controlled setting for testing SMART's mechanism in a convex online-learning problem.

Core question:

When losses are generated by a drifting target parameter $\mu_t$, can SMART stay optimistic when the process is stable and still protect against harder nonstationary phases?

This experiment is intended to be mechanistic and interpretable. It is not meant to be a real-world benchmark.

## Problem setup

The learner chooses $a_t$ in a bounded domain:

$$
a_t \in [-1,1].
$$

At round $t$, the loss is:

$$
\ell_t(a) = \tfrac{1}{2}(a-\mu_t)^2.
$$

Regret is measured against the best fixed action in hindsight on each prefix/horizon.

## Algorithms compared

- `FTL` (optimistic baseline)
- `OGD` (robust baseline)
- `SMART` (FTL prefix, one-way switch to OGD when the online signal crosses threshold)

### Robust baseline specification

The robust comparator is OGD on $[-1,1]$:

- fixed-rate option: $\eta = 2/\sqrt{n}$
- anytime option: $\eta_t = 1/\sqrt{t}$

Update is projected gradient descent for the current quadratic loss.

Why OGD is the correct robust baseline here:

1. Losses are convex and gradients are bounded on the chosen domain.
2. OGD has standard $O(\sqrt{n})$ worst-case behavior under these assumptions.
3. SMART's robust branch needs a stable no-assumption fallback; OGD is exactly that in this 1D OCO setting.

## SMART switch rule used in code

The code computes the adapted FTL trace $\Sigma_t$ (eq.6-style decomposition) and uses deterministic threshold:

$$
\text{threshold} = \texttt{threshold\_scale}\cdot 2\sqrt{n},
$$

and switches at first $t$ where

$$
\Sigma_t > \text{threshold}.
$$

After switching, the robust branch is reset and continued with OGD for the suffix.

## Sequence model ($\mu_t$) used for each scenario

All scenarios define full horizon-length sequences $\mu_1,\ldots,\mu_n$.

### 1) `stable_benign`

Goal: optimism safety.

Generator:

- AR(1)-style process around 0.60:
  - init $x_0 = 0.60 + 0.01\,\mathcal{N}(0,1)$
  - recurrence $x_t = 0.98x_{t-1} + 0.02\cdot0.60 + 0.006\,\mathcal{N}(0,1)$
- clip to $[-0.9, 0.9]$

What this regime is meant to show:

- FTL should be strong and SMART should track FTL closely.

### 2) `corruption_burst`

Goal: protection against brief severe disruptions.

Generator:

- baseline: $\mu_t = 0.55 + 0.015\,\mathcal{N}(0,1)$
- two corruption windows:
  - $[0.30n, 0.42n)$: center at $-0.85$ with $0.03\,\mathcal{N}(0,1)$ noise
  - $[0.60n, 0.72n)$: center at $-0.80$ with $0.03\,\mathcal{N}(0,1)$ noise
- clip to $[-0.95, 0.95]$

What this regime is meant to show:

- pure optimism can overcommit during burst windows; SMART should gain by switching.

### 3) `drift_plus_shift`

Goal: representative mixed nonstationarity.

Generator:

- early benign phase up to $0.55n$: $0.55 + 0.012\,\mathcal{N}(0,1)$
- gradual downward drift on $[0.55n, 0.72n)$
- sustained negative shift on $[0.72n, n)$: $-0.70 + 0.02\,\mathcal{N}(0,1)$
- clip to $[-0.95, 0.95]$

What this regime is meant to show:

- SMART should be a pragmatic middle policy when the environment transitions from easy to hard.

## Evaluation protocol

Primary figure protocol is regret-vs-horizon:

- horizons: $n = 100, 200, \ldots, 1000$
- for each horizon, draw fresh sequences of exactly that length
- default trials per horizon: 30
- report mean regret and 95% CI ($\text{mean} \pm 1.96\,\text{SE}$)

This is intentionally not a single-run prefix trace.

## Current empirical interpretation

From the curated figure (`fig_exp03_quadratic_oco_regret_by_horizon.png`), the intended narrative is visible:

1. `stable_benign`: SMART is near FTL and both are below robust OGD.
2. `corruption_burst`: SMART reduces damage relative to pure FTL as burst impact accumulates.
3. `drift_plus_shift`: SMART stays between optimistic and robust extremes in a way consistent with mixed-regime adaptation.

## What this experiment supports

This experiment provides controlled evidence that SMART's one-switch mechanism carries to convex losses:

- no unnecessary tax in benign dynamics,
- useful protection under structured nonstationarity,
- interpretable behavior through explicit switch timing.

## Limits and non-claims

- Sequences are synthetic constructions, not fitted from observed data.
- Threshold scaling (`--threshold-scale`, default `0.0035`) is an explicit modeling choice.
- Conclusions are about mechanism behavior in this toy-convex setting, not universal performance guarantees.
